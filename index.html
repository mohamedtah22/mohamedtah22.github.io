Operating Systems

xv6 kernel extensions

xv6 Shared Memory & Multi-process Logging System

C • xv6 Kernel • Virtual Memory • Atomic Sync

xv6

Extended the xv6 operating system with shared memory support and used it to build a synchronized multi-process logging system. The project enables multiple processes to share physical pages safely and coordinate access to a shared logging buffer using atomic operations.

  • Shared Memory API: implemented map_shared_pages to map physical pages across processes
  • VM work: used walk() and mappages(); handled alignment with PGROUNDDOWN/PGROUNDUP
  • Safety: introduced a custom shared-page flag (PTE_S) so pages are freed only by original owners
  • Logging: shared buffer + atomic CAS (__sync_val_compare_and_swap) to prevent races
  • Tests: shmem_test and log_test + run via make qemu

Types

Extended the L5 functional language into L52 by implementing an advanced static type-checker and an interpreter. The project focuses on set-theoretic types and type reasoning inside programs.

  • Complex types: Union, Intersection, and Difference types (set-theoretic principles)
  • Type predicates: support for predicates (e.g., is? number) with type narrowing
  • Normalization: flattening/normalizing into Disjunctive Normal Form (DNF)
  • Subtyping: extended subtype logic for special atomic types (e.g., any, never)
  • Testing: run with npm test
